# Managing Services on Fedora

Linux uses services (also called daemons) to run background processes such as SSH, networking, printing, databases, and web servers.

Fedora uses **systemd** as its init system and **systemctl** as the primary tool for managing services.

---

# What is a Service?

A service is a program that runs in the background and provides functionality to the operating system or applications.

Examples:

* SSH Server (`sshd`)
* Network Manager (`NetworkManager`)
* Printing Service (`cups`)
* Web Servers (`httpd`, `nginx`)
* Database Servers (`mysql`, `postgresql`)

---

# What is systemd?

systemd is the system and service manager used by Fedora.

It is responsible for:

* Starting services during boot
* Stopping services
* Monitoring services
* Managing system states

---

# What is systemctl?

`systemctl` is the command-line utility used to interact with systemd.

### Syntax

```bash
systemctl <command> <service>
```

Example:

```bash
sudo systemctl status sshd
```

---

# View Service Status

Display the current status of a service.

### Syntax

```bash
systemctl status <service>
```

### Example

```bash
systemctl status sshd
```

Example Output:

```text
● sshd.service - OpenSSH Server Daemon
   Loaded: loaded
   Active: active (running)
```

---

# Start a Service

Start a service immediately.

### Syntax

```bash
sudo systemctl start <service>
```

### Example

```bash
sudo systemctl start sshd
```

---

# Stop a Service

Stop a running service.

### Syntax

```bash
sudo systemctl stop <service>
```

### Example

```bash
sudo systemctl stop sshd
```

---

# Restart a Service

Restart a service after configuration changes.

### Syntax

```bash
sudo systemctl restart <service>
```

### Example

```bash
sudo systemctl restart sshd
```

---

# Reload a Service

Reload configuration without fully restarting the service.

### Syntax

```bash
sudo systemctl reload <service>
```

### Example

```bash
sudo systemctl reload sshd
```

---

# Enable a Service

Enable a service to start automatically during boot.

### Syntax

```bash
sudo systemctl enable <service>
```

### Example

```bash
sudo systemctl enable sshd
```

Verify:

```bash
systemctl is-enabled sshd
```

Output:

```text
enabled
```

---

# Disable a Service

Prevent a service from starting automatically during boot.

### Syntax

```bash
sudo systemctl disable <service>
```

### Example

```bash
sudo systemctl disable sshd
```

---

# Check if a Service is Running

### Syntax

```bash
systemctl is-active <service>
```

### Example

```bash
systemctl is-active sshd
```

Output:

```text
active
```

---

# List Running Services

Display all active services.

### Command

```bash
systemctl list-units --type=service
```

---

# List Failed Services

Display services that failed to start.

### Command

```bash
systemctl --failed
```

Example Output:

```text
UNIT            LOAD   ACTIVE SUB    DESCRIPTION
example.service loaded failed failed Example Service
```

---

# View Service Logs

Use journalctl to view logs generated by a service.

### Syntax

```bash
journalctl -u <service>
```

### Example

```bash
journalctl -u sshd
```

---

# View Recent Service Logs

Display the most recent entries.

### Example

```bash
journalctl -u sshd -n 20
```

---

# Follow Service Logs in Real Time

Similar to:

```bash
tail -f
```

### Example

```bash
journalctl -u sshd -f
```

# Common Problems

## Service Not Found

Example:

```text
Unit sshd.service could not be found
```

### Cause

The package providing the service is not installed.

### Solution

Install the required package.

---

## Service Failed to Start

Check logs:

```bash
journalctl -u service_name
```

View recent errors:

```bash
systemctl status service_name
```

---

## Service Starts But Stops Immediately

Possible causes:

* Incorrect configuration
* Missing dependencies
* Permission issues

Check logs:

```bash
journalctl -xe
```
